Memory & Storage Systems
Primary Memory, Secondary Memory, RAM, ROM, Cache, Memory Hierarchy

Primary Memory

Definition + Types ▼
Definition: Primary memory (also called main memory) is the memory directly accessible by the CPU. It stores data and instructions currently in use.
Key Characteristics
  • Directly accessed by the CPU without any intermediate steps
  • Faster than secondary memory but smaller in capacity
  • Volatile (RAM) — loses data when power is off, or Non-volatile (ROM)
  • Also called internal memory or main memory
Types of Primary Memory
RAMRead/Write, Volatile, used for running programs
ROMRead-only, Non-volatile, stores firmware/BIOS
CacheUltra-fast, between CPU and RAM, stores recent data
RegistersInside CPU, fastest storage, holds current instructions

Secondary Memory

Definition + Examples ▼
Definition: Secondary memory is non-volatile, permanent storage used to store data and programs that are not currently in use by the CPU.
  • Cannot be directly accessed by the CPU — data must first be loaded into primary memory
  • Much larger capacity than primary memory but much slower
  • Non-volatile — retains data without power
  • Also called external memory or auxiliary memory
Examples
HDD (Hard Disk Drive)Magnetic storage, large capacity, slower
SSD (Solid State Drive)Flash-based, fast, no moving parts
USB Flash DrivePortable, plug-and-play, flash memory
Optical DiscsCD, DVD, Blu-ray — laser-read
Comparison: Primary vs Secondary

Volatile?Yes (RAM)No

RAM — Random Access Memory

Volatile ▼
Definition: RAM is a volatile primary memory that allows data to be read and written in any order. It temporarily stores data and programs currently in use by the CPU.
  • Volatile — all data is lost when the computer is turned off
  • Read and write operations are both possible
  • The more RAM a computer has, the more programs it can run simultaneously
  • Modern computers typically have 4GB to 64GB of RAM
Types of RAM
TypeFull FormKey Feature
DRAMDynamic RAMNeeds constant refreshing; used as main memory
SRAMStatic RAMNo refreshing needed; faster; used in cache
SDRAMSynchronous DRAMSynchronized with CPU clock speed
DDR RAMDouble Data RateTransfers data twice per clock cycle; DDR4/DDR5 today

ROM — Read Only Memory

Non-Volatile ▼
Definition: ROM is a non-volatile primary memory whose content is permanently written during manufacturing. Data cannot be modified (or is very difficult to modify) under normal operation.
  • Non-volatile — retains data even when power is off
  • Stores firmware — instructions that run when the computer starts (BIOS/UEFI)
  • Typically read-only during normal operation
Types of ROM
TypeFull FormDescription
ROMRead Only MemoryProgrammed at factory; cannot be changed

EPROMErasable PROMErased with UV light; reprogrammable

Cache Memory

Fastest Memory ▼
Definition: Cache memory is a small, extremely fast memory located between the CPU and RAM. It stores frequently used data and instructions so the CPU can access them quickly without waiting for RAM.
  • Much faster than RAM but much smaller in size
  • Reduces the time the CPU spends waiting for data
  • Uses SRAM technology (Static RAM)
  • Managed automatically by hardware
Cache Levels
LevelLocationSizeSpeed
L1 CacheInside CPU core32–512 KBFastest (1–4 cycles)
L2 CacheNear CPU core256 KB–4 MBVery fast
L3 CacheShared across cores4 MB–64 MBFast but slower than L1/L2

Memory Hierarchy

Diagram Topic ▼
Definition: Memory hierarchy is an arrangement of different types of computer storage organized by speed, cost, and capacity — from fastest/smallest/most expensive at the top to slowest/largest/cheapest at the bottom.
The Hierarchy (Top to Bottom = Fastest to Slowest)
① Registers — Inside CPU, nanoseconds, bytes in size
② Cache Memory (L1→L2→L3) — KB to MB
③ RAM (Primary Memory) — GB range, milliseconds
④ SSD / HDD (Secondary Memory) — TB range, seconds
⑤ Cloud / Tape / Optical — Archival storage
  • As you go up the hierarchy: speed increases, cost increases, capacity decreases
  • As you go down the hierarchy: speed decreases, cost decreases, capacity increases
  • The CPU first checks registers → cache → RAM → secondary storage
  • This hierarchy is designed to balance performance and cost effectively

Modem — Why It Is Needed & Why It Is a Peripheral

Important Short Answer ▼
Definition: A modem (Modulator-Demodulator) is a device that converts digital signals from a computer into analog signals for transmission over telephone/cable lines and vice versa.
Why Is a Modem Needed?
  • Computers communicate using digital signals (0s and 1s)
  • Telephone and cable networks transmit analog signals (continuous waves)
  • The modem MODulates (digital → analog) when sending data
  • The modem DEModulates (analog → digital) when receiving data
  • Without a modem, a computer cannot connect to the internet via telephone/cable lines
Why Is a Modem a Peripheral Device?
  • A peripheral device is any device connected to a computer that is not part of the core processing unit (CPU, RAM, motherboard)
  • A modem is externally attached — it can be internal (PCI card) or external (USB/standalone box)
  • It is not essential for the computer to function — the computer works without it
  • It enhances functionality (internet access) but is not a core component
  • Therefore it qualifies as a peripheral input/output device

Number System Conversions
Dec↔Bin↔Oct↔Hex, BCD method, Multiplication, Division, 2's Complement

Decimal → Binary, Octal, Hexadecimal

Conversion ▼
Decimal to Binary (Base 10 → Base 2)
Method: Divide by 2 repeatedly, read remainders bottom to top.
Example: 25 (decimal) → Binary 25 ÷ 2 = 12 remainder 1 12 ÷ 2 = 6 remainder 0 6 ÷ 2 = 3 remainder 0 3 ÷ 2 = 1 remainder 1 1 ÷ 2 = 0 remainder 1 Read bottom to top → 11001 ∴ 25 (decimal) = 11001 (binary)
Decimal to Octal (Base 10 → Base 8)
Method: Divide by 8 repeatedly, read remainders bottom to top.
Example: 125 → Octal 125 ÷ 8 = 15 remainder 5 15 ÷ 8 = 1 remainder 7 1 ÷ 8 = 0 remainder 1 Read bottom to top → 175 ∴ 125 (decimal) = 175 (octal)
Decimal to Hexadecimal (Base 10 → Base 16)
Method: Divide by 16 repeatedly. Remainders 10–15 = A–F.
Example: 255 → Hex 255 ÷ 16 = 15 remainder 15 = F 15 ÷ 16 = 0 remainder 15 = F Read bottom to top → FF ∴ 255 (decimal) = FF (hex)

Binary, Octal, Hex → Decimal

Conversion ▼
Method for all: Multiply each digit by its base raised to its position power, then add all results.
Binary to Decimal
Example: 11001 (binary) → Decimal = 1×2⁴ + 1×2³ + 0×2² + 0×2¹ + 1×2⁰ = 16 + 8 + 0 + 0 + 1 = 25
Octal to Decimal
Example: 175 (octal) → Decimal = 1×8² + 7×8¹ + 5×8⁰ = 64 + 56 + 5 = 125
Hexadecimal to Decimal
Example: 2F (hex) → Decimal = 2×16¹ + F×16⁰ = 32 + 15 = 47

Octal ↔ Hex using Binary as Bridge

Via Binary ▼
Key Concept: There is no direct Octal↔Hex formula. Convert to Binary first (as bridge), then convert Binary to the target base.
Octal → Hex (Octal → Binary → Hex)
Step 1 Convert each Octal digit to 3-bit Binary
Step 2 Join all binary bits together
Step 3 Group binary bits into groups of 4 from right
Step 4 Convert each 4-bit group to Hex digit
Example: 175 (octal) → Hex Step 1: 1→001, 7→111, 5→101 Step 2: 001 111 101 Step 3: Group in 4s from right: 0111 1101 Step 4: 0111=7, 1101=D ∴ 175 (octal) = 7D (hex)
Hex → Octal (Hex → Binary → Octal)
Step 1 Convert each Hex digit to 4-bit Binary
Step 2 Join all binary bits together
Step 3 Group binary bits into groups of 3 from right
Step 4 Convert each 3-bit group to Octal digit
Example: 7D (hex) → Octal Step 1: 7→0111, D→1101 Step 2: 0111 1101 Step 3: Group in 3s from right: 001 111 101 Step 4: 001=1, 111=7, 101=5 ∴ 7D (hex) = 175 (octal)
Important: Add leading zeros if the last group on the left has fewer than 3 (or 4) bits.

Binary Multiplication & Division

Arithmetic ▼
Binary Multiplication Rules
0 × 0 = 0Zero times zero is zero
0 × 1 = 0Zero times one is zero
1 × 0 = 0One times zero is zero
1 × 1 = 1One times one is one
Binary multiplication works just like decimal — multiply each bit of the multiplier by the multiplicand, shift left, then add.
Example: 1011 × 101 1011 (multiplicand) × 101 (multiplier) ------ 1011 (1011 × 1, shift 0) 0000 (1011 × 0, shift 1) 1011 (1011 × 1, shift 2) -------- 110111 = 110111 (binary) Verify: 11 × 5 = 55 = 110111 ✓
Binary Division
Binary division follows the same long-division method as decimal. Work with 0 and 1 only.
Example: 1100 ÷ 11 (12 ÷ 3 = 4) 100 ------ 11 | 1100 11 ------ 00 00 ------ 0 ∴ Quotient = 100 (binary) = 4, Remainder = 0

Binary Subtraction using 2's Complement

Very Important ▼
Definition: 2's Complement is a method to represent negative binary numbers and perform subtraction using addition. To subtract A – B, we add A + (2's complement of B).
How to Find 2's Complement
Step 1 Write the binary number
Step 2 Find 1's complement — flip all bits (0→1, 1→0)
Step 3 Add 1 to the 1's complement result
Example: 2's complement of 0110 (6) Step 1: 0110 Step 2: 1's complement = 1001 (flip all bits) Step 3: 1001 + 1 = 1010 (2's complement)
Binary Subtraction: A – B using 2's Complement
Step 1 Find 2's complement of B
Step 2 Add A + (2's complement of B)
Step 3 If carry out occurs → discard carry → result is positive. If no carry → take 2's complement of result → that is the answer (negative)
Example: 1101 – 0101 (13 – 5 = 8) Step 1: 2's complement of 0101 1's comp = 1010 +1 = 1011 Step 2: 1101 + 1011 ------ 11000 ← carry out (5 bits) Step 3: Discard carry → 1000 = 8 ✓
Example: 0101 – 1101 (5 – 13 = –8) 2's comp of 1101 = 0011 0101 + 0011 ------ 1000 (no carry out → result is negative) 2's comp of 1000 = 1000 → answer is –1000 = –8 ✓

Social Impact of Computers

Positive & Negative ▼
Definition: The social impact of computers refers to the ways in which computer technology has changed human society — including education, work, communication, healthcare, and ethics.
Positive Impacts
  • Education: E-learning, online courses, digital libraries make education accessible globally. Students in remote areas like Lalmanirhat can access world-class resources
  • Communication: Email, video calls, social media enable instant global communication at near-zero cost
  • Healthcare: Computer-aided diagnosis, digital records, telemedicine, MRI/CT scan processing improves patient care
  • Business & Economy: E-commerce, digital banking, automation increase productivity and create new industries
  • Scientific Research: Simulation, big data analysis, and AI accelerate discoveries in medicine, climate, and space
  • Entertainment: Streaming, gaming, digital art and music production create entirely new creative industries
  • Government Services: e-Government, digital IDs, online tax filing make services faster and more transparent
Negative Impacts
  • Unemployment: Automation replaces many manual and routine jobs in manufacturing, data entry, and customer service
  • Privacy Concerns: Personal data collection, surveillance, and hacking threaten individual privacy
  • Cybercrime: Online fraud, identity theft, ransomware, and phishing attacks cause billions in losses
  • Digital Divide: Gap between those with and without computer/internet access widens social inequality
  • Health Issues: Excessive screen time causes eye strain, poor posture, sleep disorders, and social isolation
  • Misinformation: Social media spreads fake news and disinformation rapidly, affecting elections and public health
  • Environmental Impact: E-waste from discarded devices and high energy consumption of data centers harm the environment
Exam tip: Write at least 4 positive and 4 negative impacts. Always give one example per point for full marks.
